## phase6 예비 추천

**v1 Stacked** 을 기본 후보로 제안. 이유:
1. 구현 복잡도 최저, Karpathy "Simplicity First" 정합.
2. 디자인 시스템 확장 0 (기존 카드/폼 토큰 재사용).
3. 태블릿 baseline (갤탭 S6 Lite) 에서 가장 안정.
4. MVP 범위 (문항 10개 상한) 에서 스크롤 길이 수용 가능.
5. Kahoot/Quizizz 사용자 학습 전이 최대.

**v2** 는 보기 길이가 짧은(<15자) 경우 매력적 — 현재 LLM 생성 보기는 평균 20자라 부적합.
**v3** 는 문항 20개+ 에서 고려 — 상한은 20이지만 기본이 10이고 재미용 20문항은 빈도 낮음 → v1 유지 타당성 변함없음.
**v4** 는 드래그 재정렬이 IN 되면 재평가.

선택 권한은 phase6 design_reviewer 에.

# Scope Decision — statistics-class-board

## 1. 선택한 UX 패턴

`phase1/ux_patterns.json`에서 7개 패턴을 분석한 결과, 아래와 같이 채택한다.

| 패턴 | 결정 | 사유 |
|---|---|---|
| 브레이크아웃룸 | **ADOPT** | 기존 코드베이스에 이미 구현되어 있으므로 인프라 재활용. 팀별 데이터 격리와 메인 보드 오버뷰를 동시에 만족(research_pack.md §2 참고). |
| 단계별 미션 가이드 | **ADAPT** | 11단계를 수직 스텝퍼로 표시하되, 현재 미션만 확장하고 이전/다음은 요약 카드로 표시(research_pack.md §4, §7 참고). 병렬 작업은 허용하지 않고 선형 진행으로 단순화. |
| 교사 승인 게이트 | **ADOPT** | 요구사항 핵심. Google Classroom의 '제출→승인→반환' 플로우를 참고하되, iterative 사이클(재수정→재승인) 추가(research_pack.md §3 참고). |
| 상태 기반 보드 트래킹 | **ADOPT** | Linear의 상태 파이프라인 개념을 차용. 5단계 상태(⬜시작전 🟡수정중 🟢승인완료 🔵교사제작중 ✅완료)를 미션별로 관리(research_pack.md §4 참고). |
| 질문 사다리 입력 폼 | **ADAPT** | Notion의 인라인 AI UX와 결합. 아코디언 형태로 한 단계씩 확장, 예시 문구는 풍선 도움말로 제공(research_pack.md §5 참고). |
| LLM 피드백 인라인 | **ADOPT** | 기존 아우라보드 API를 활용. 교사 승인과 병행하여 LLM은 '조언' 수준으로만 작동(research_pack.md §6 참고). |
| 포스터 의뢰서 제출 폼 | **ADOPT** | 미션 1~8 산출물을 자동 연동하여 중복 입력 최소화(research_pack.md §7 참고). |

## 2. MVP 범위

### 포함 (IN)

1. **브레이크아웃룸 내 팀별 미션 보드**
   - 기존 브레이크아웃룸을 확장하여 팀별 미션 공간 제공
   - 팀원 접근 권한 제어 (팀원/교사만 접근)

2. **11단계 미션 템플릿 자동 생성**
   - 학급보드 생성 시 11개 미션 자동 생성
   - 각 미션별 산출물 템플릿 제공

3. **질문 사다리 입력 폼 (미션 2)**
   - 6단계 아코디언: 경험→현황→이유→조건→대안→입장
   - 단계별 예시 문구 및 도움말
   - 단계별 완료 체크

4. **교사 승인 게이트 (미션 2, 3, 6, 8)**
   - 학생 '완료' 표시 → 승인 요청 상태로 전환
   - 교사 승인 전 다음 미션 잠금
   - 승인/반려(피드백) 기능
   - 반려 시 수정 가능, 재승인 요청 가능

5. **팀별 미션 상태 트래킹**
   - 5단계 상태: ⬜시작전 🟡수정중 🟢승인완료 🔵교사제작중 ✅완료
   - 학생용 수직 스텝퍼 UI
   - 교사용 대시보드(표 형태)

6. **LLM 피드백 연동**
   - 질문 사다리 입력 시 'AI 조언 받기' 버튼
   - 기존 API 호출, 3초 내 응답 표시
   - 피드백은 참고용으로만 표시 (교사 승인이 최종)

7. **포스터 제작 의뢰서 (미션 9)**
   - 정형화된 폼: 팀명, 주제, 데이터, 결론, 제안 등
   - 미션 1~8 산출물 자동 연동
   - 의뢰서 제출 시 교사에게 알림(배지)

8. **기본 권한 모델**
   - 교사(Teacher): 모든 팀 보드 접근, 승인/반려, 대시보드
   - 학생(Student): 소속 팀 보드만 접근, 미션 수행, 의뢰서 제출

### 제외 (OUT)

| 기능 | 제외 이유 | 후속 task 예정 |
|---|---|---|
| **설문 배포 및 응답 수집** | 설문 링크 생성, 응답 DB 스키마, 통계 계산 등 별도 도메인. MVP에서는 설문 문항 텍스트 작성까지만 지원. | `feature/survey-distribution` |
| **그래프 생성/렌더링** | 차트 라이브러리(Chart.js, D3) 도입 및 데이터 바인딩 복잡도. MVP에서는 그래프 종류/제목 계획까지만 지원. | `feature/mission-chart-builder` |
| **포스터 에디터/제작 도구** | 교사가 포스터를 제작하는 에디터 UI는 별도 큰 기능. MVP에서는 의뢰서 제출까지만 지원. | `feature/poster-editor` |
| **푸시 알림/이메일** | 실시간 푸시(웹푸시/FCM) 인프라 필요. MVP에서는 UI 내 배지/토스트 알림만. | `feature/notification-system` |
| **파일/이미지 업로드** | 그래프 이미지 첨부, 사진 기록 등 스토리지 인프라 필요. MVP에서는 텍스트/URL만 입력. | `feature/file-upload` |
| **실시간 동시 편집** | 기존 padlet WebSocket은 카드 단위. 미션 폼 단위 실시간 충돌 처리 복잡. MVP에서는 낙관적 업데이트 + 수동 새로고침. | `feature/realtime-mission-sync` |
| **타이머/투표 기능** | Miro의 브레이크아웃룸 타이머/투표는 교육 도구로 유용하나 MVP 범위 초과. | `feature/classroom-tools` |

## 3. 수용 기준 (Acceptance Criteria)

1. 교사가 학급보드를 생성하면 11개 미션이 자동으로 생성되고, 각 팀의 브레이크아웃룸에 할당된다.
2. 학생이 팀에 참여하면 해당 팀의 미션 보드에 접근할 수 있고, 다른 팀 보드는 보이지 않는다.
3. 미션 2(질문 사다리)에서 6단계(경험→현황→이유→조건→대안→입장) 입력 필드가 아코디언 형태로 순서대로 노출된다.
4. 학생이 미션 2를 '완료'로 표시하면 상태가 🟡수정중에서 승인요청으로 변경되고, 교사가 승인하기 전까지 미션 3의 '시작' 버튼은 비활성화된다.
5. 교사가 승인하면 해당 팀의 미션 2 상태가 🟢승인완료로 변경되고, 미션 3의 잠금이 해제된다.
6. 질문 사다리 입력 영역에서 'AI 조언 받기' 버튼을 클릭하면 기존 API를 호출하고, 3초 이내에 피드백 텍스트가 입력 영역 하단에 표시된다.
7. 미션 9(포스터 의뢰서) 폼에 미션 1(주제), 미션 2(질문사다리), 미션 3(설문문항) 등의 산출물이 자동으로 채워진다.
8. 교사용 대시보드(`/dashboard`)에서 모든 팀의 현재 미션 번호와 상태가 표 형태로 표시되며, 상태별 필터링이 가능하다.
9. 학생용 보드에서 현재 팀의 11개 미션이 수직 스텝퍼로 표시되고, 현재 미션만 확장되어 입력 폼이 노출된다.
10. 미션 8(결론·제안·한계점)이 🟢승인완료 상태가 되면 미션 9(포스터 의뢰서)의 잠금이 자동으로 해제된다.

## 4. 스코프 결정 모드

**Selective Expansion**

핵심 워크플로우(11미션 + 질문사다리 + 승인게이트 + LLM피드백 + 의뢰서)는 모두 포함하지만, 설문 배포/그래프 렌더링/포스터 에디터/푸시알림 등은 후속 task로 분리한다. 기존 브레이크아웃룸과 LLM API 인프라를 재활용하여 신규 개발 범위를 최소화한다.

## 5. 위험 요소

| 위험 | 영향도 | 완화책 |
|---|---|---|
| **LLM API 응답 지연/실패** | 높음 | 타임아웃 5초 설정. 실패 시 "잠시 후 다시 시도해 주세요" 폴백. API 호출은 비동기로, 입력 필드는 차단하지 않음. |
| **교사 승인 지연으로 진행 정체** | 높음 | 승인 대기 중에도 이전 미션은 수정 가능(재제출). 교사용 대시보드에 '승인 대기 중' 팀을 상단 고정. |
| **11단계 미션의 학생 인지 부하** | 중간 | 현재 미션 ±1만 기본 노출, 나머지는 '더 보기'로 접기. 각 미션 예상 소요 시간 안내. |
| **팀별 데이터 격리 누수** | 중간 | 기존 브레이크아웃룸 권한 모델을 그대로 확장. API 엔드포인트마다 팀 멤버십 검증 미들웨어 필수. |
| **모바일에서 질문 사다리 입력 불편** | 중간 | 아코디언 대신 하단 탭 네비게이션으로 단계 전환(모바일 전용). 텍스트area는 자동 확장. |
| **상태 전환 경쟁 조건** | 낮음 | 낙관적 업데이트 사용 시 동시 상태 변경 충돌 가능. 버전 필드(etag)로 낙관적 잠금 적용. |

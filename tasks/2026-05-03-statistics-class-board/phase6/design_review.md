## 수정 상세

### 1. 반응형 전환 규칙 보강 (768~1023px)

**문제**: 초기 스펙에서 768~1023px 구간의 처리가 모호했음 (v1 축소 vs v2 rail 중 불명확).

**수정**:
- 768~1023px: 사이드바 220px 축소 유지 (제목 12px, 상태 캡션 숨김, 숫자 원 24px)
- 860px 이하: textarea min-height 60px로 축소
- <768px: 사이드바 완전 숨김, 상단 고정 탭 + 하단 고정 액션바

**근거**: v2 Minimal Rail은 초등학생에게 직관성이 떨어지므로, 사이드바를 축소하되 텍스트는 유지하는 것이 최적.

### 2. 접근성 보강 (모바일 탭 + LLM 로딩)

**문제**: 모바일 가로 스크롤 탭의 키보드 접근성과 LLM 로딩 상태의 스크린리더 지원이 미흡.

**수정**:
- 모바일 탭: `role="tablist"`, `aria-orientation="horizontal"`, 좌우 화살표 키 전환, 활성 탭 외 `tabindex="-1"`
- LLM 로딩: `role="status"`, `aria-live="polite"`로 "AI 조언을 불러오는 중" / "AI 조언이 도착했습니다" 안내
- 반려 상태: `aria-live="assertive"`로 즉시 알림

### 3. 교사 대시보드 정보 계층 추가

**문제**: 교사용 대시보드의 시각적/정보적 계층이 구체적으로 정의되지 않음.

**수정**:
- 상단: "승인 대기 N건" 카운트 배너 (`--color-status-pending-bg`)
- 중앙: 팀별 진행 표 (팀명 | 현재 미션 | 상태)
- 하단/우측: 선택한 팀의 상세 승인 패널 (`SidePanel` 재활용)

### 4. 모바일 상태 설계 추가

**문제**: design_spec에 모바일 상태(F)가 누락되어 있었음.

**수정**:
- "상태 F: mobile ready" 추가
- 상단 탭 바: 44px, scroll-snap, safe-area-inset-bottom 고려
- 하단 액션바: fixed, safe-area 적용

## AI Slop 감지 결과

- ❌ **기계적 반복 패턴 없음**: 각 상태별 디자인이 기능적 맥락에 맞게 차별화됨
- ❌ **무의미한 그라디언트 없음**: 디자인 시스템 규칙 준수, 그라디언트 미사용
- ❌ **Placeholder 텍스트 없음**: 모든 UI 요소가 실제 기능/데이터와 연결됨
- ❌ **과도한 그림자/효과 없음**: `--shadow-card`, `--shadow-accent`만 사용
